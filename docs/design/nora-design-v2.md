# Nora UI/UX 重构设计规格

> 版本：v2.0 | 日期：2026-04-25 | 状态：草案

---

## 1. 设计愿景

**"一键即聊，无感切换"**

用户打开 Nora，无需任何配置，直接开始对话。模型在后台默默加载，用户只看到结果。所有复杂的技术细节被优雅地隐藏在简洁界面之下。

**核心原则：**
1. **零摩擦进入** — 开屏即对话，无 onboarding 阻碍
2. **渐进式引导** — 用功能卡片代替文字说明
3. **安静界面** — 技术细节被优雅隐藏，只展示对话本身
4. **AI Native 视觉** — 科技感 + 温暖感的平衡

## 2. 对标分析

### 豆包（字节跳动）
| 维度 | 豆包策略 | Nora 借鉴 |
|------|----------|-----------|
| 首屏 | Logo + 欢迎语 + 快捷卡片 | ✅ 采用 |
| 输入框 | 底部居中，占据 40% 宽度 | ✅ 采用 |
| 模型选择 | 隐藏在下拉菜单 | ❌ 移除，AI 自主决策 |
| 快捷功能 | 2-3 张功能卡片 | ✅ 采用 |
| 历史对话 | 左滑显示 | ✅ 采用（侧边栏） |

### ChatGPT（OpenAI）
| 维度 | ChatGPT 策略 | Nora 借鉴 |
|------|--------------|-----------|
| 视觉风格 | 深色极简，无干扰 | ✅ 采用 |
| 欢迎语 | 个性化 + 能力展示 | ✅ 采用 |
| 输入框 | 大且居中，圆角胶囊 | ✅ 采用 |
| 流式输出 | 打字机效果 + 光标 | ✅ 保留 |
| 设置入口 | 左下角图标 | ✅ 采用（更隐蔽） |

### 差异化的机会
- **完全离线** — 可以强调"数据永不离开设备"作为信任背书
- **中国特色** — 通知聚合、本地文件上下文（豆包优势）
- **轻量化** — 不需要账号体系，真正的零配置

## 4. 核心组件规范

### 4.1 TopBar（顶部栏）

| 元素 | 样式 | 交互 |
|------|------|------|
| Logo/标题 | NoraOrange (#FF6B6B) + 20sp Bold | 点击 → 侧边栏 |
| 状态指示器 | 绿点=就绪，黄点=加载中 | 无交互 |
| 菜单按钮 | Icon only, 24dp | 点击 → 通用菜单 |
| 设置按钮 | Icon only, 24dp | 点击 → 设置页 |

**设计原则：** 高度 ≤ 56dp，背景与页面一致，不做分割线。

### 4.2 欢迎区块（空状态）

**首次用户看到：**
```
┌─────────────────────────────────────┐
│                                     │
│     ◉ Nora                          │  ← 品牌图标（发光效果）
│                                     │
│   欢迎回来，我是 Nora                │  ← 首次：欢迎使用
│                                     │     回访：欢迎回来
│   你的本地 AI 助手                  │
│   数据永不离开你的设备               │  ← 信任背书（小字）
│                                     │
└─────────────────────────────────────┘
```

**设计细节：**
- 上下居中，与输入框保持 48dp 间距
- 品牌图标：48dp，带 subtle glow 效果
- 文字颜色：TextPrimary
- 信任背书：TextSecondary，12sp

### 4.3 快捷功能卡片

**布局：** 横向滚动，2-3 张卡片

| 卡片 | 图标 | 描述 |
|------|------|------|
| 读文件 | 📁 | 加载本地文件作为上下文 |
| 看通知 | 🔔 | 聚合设备通知并摘要 |
| 写代码 | 💻 | 代码生成与解释 |
| 翻译 | 🌐 | 文档翻译 |

**样式：**
- 卡片尺寸：120dp × 100dp
- 圆角：16dp
- 背景：SurfaceEdge (#2C2C2C)
- 阴影：无（保持平面感）
- 间距：12dp

**交互：**
- 点击 → 执行对应功能
- 长按 → 显示功能详情（可选）

### 4.4 消息气泡

**用户消息：**
```
背景：NoraOrange (#FF6B6B)
圆角：16dp（四周）
最大宽度：屏幕宽度 × 75%
位置：右对齐
间距：消息间 8dp，气泡内 10dp 14dp
```

**AI 回复：**
```
背景：SurfaceEdge (#2C2C2C)
圆角：16dp（四周）
最大宽度：屏幕宽度 × 80%
位置：左对齐
头像：NoraLogo 小号（可选）
打字机效果：每字符 20ms
流式光标：▊ 颜色 TextSecondary
```

**错误消息：**
```
背景：ErrorContainer
文字：Error
图标：⚠️
```

### 4.5 输入框

**设计原则：** 大、醒目、零门槛

```
高度：56dp（含按钮）
圆角：28dp（胶囊形）
背景：SurfaceEdge
边框：无（纯背景）
placeholder：TextSecondary，"发送消息给 Nora..."
```

**发送按钮：**
- 位置：输入框右侧内
- 图标：ArrowUp（不是 Send）
- 状态：
  - 空输入：灰色，不可点击
  - 有输入：NoraOrange，可点击
  - 发送中：旋转动画

**进阶：语音输入图标（未来特性）**

### 4.6 侧边栏

**入口：**
- 点击 TopBar 左侧 Logo
- 右滑（可选）

**结构：**
1. 头部：Logo + "Nora"
2. 新建对话按钮（Primary 样式）
3. 历史对话列表（按天分组）
4. 底部：设置入口 + 模型状态

**模型状态指示器：**
```
🟢 已就绪     — 模型已加载
🟡 加载中     — 显示进度百分比
🔴 未加载     — 点击重试
```

## 5. 渐进式设计

### Phase 1：极简对话（当前）
- 移除所有干扰元素
- 只有一个输入框 + 欢迎语
- 历史记录在侧边栏

### Phase 2：快捷功能（下一个 Sprint）
- 添加 2-3 张功能卡片
- 卡片触发对应能力
- 文件选择器集成

### Phase 3：上下文感知（未来）
- 根据对话内容智能推荐功能
- 通知摘要自动触发
- 文件上下文自动关联

## 6. 暗色主题规范

**保留现有的 NoraOrange 作为唯一暖色**

| Token | 值 | 用途 |
|-------|-----|------|
| Background | #121212 | 页面背景 |
| Surface | #1E1E1E | 卡片、对话框 |
| SurfaceEdge | #2C2C2C | 输入框、次级卡片 |
| NoraOrange | #FF6B6B | 主 CTA、品牌色 |
| TextPrimary | #E0E0E0 | 正文 |
| TextSecondary | #9E9E9E | 次要文字 |
| MatrixGreen | #00FF41 | 可选：Hacker 模式 |

**不跟随系统主题，强制暗色**

## 7. 动画规范

| 场景 | 动画 | 时长 |
|------|------|------|
| 消息出现 | alpha 0→1 | 200ms ease-out |
| 流式输出 | 逐字显示 | 20ms/字符 |
| 发送按钮 | 旋转 | 1000ms linear |
| 侧边栏滑入 | translateX | 250ms ease-out |
| 快捷卡片点击 | scale 0.95 | 100ms |
| 加载指示器 | pulse | 1500ms infinite |

---

## 8. 移除项清单

❌ 模型选择下拉框（AI 自主决策）  
❌ 设置页面中的"模型路径"（内部细节）  
❌ 专业术语展示（如"ExecuTorch"、"Qwen3"）  
❌ 独立的"加载模型"按钮（自动后台加载）  
❌ 复杂的权限引导页（渐进式请求）
